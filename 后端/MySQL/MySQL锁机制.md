# MySQL 锁机制

> 来源：[MySQL 有哪些锁？](https://xiaolincoding.com/mysql/lock/mysql_lock.html)
> 一句话总结：MySQL 按加锁范围分为全局锁、表级锁、行级锁，InnoDB 主要通过细粒度的行级锁与 MVCC 兼顾并发与隔离。

## 一、锁的分类总览

![MySQL 锁分类](images/mysql-lock-overview.png)

| 分类维度 | 锁类型 | 加锁范围 | 主要用途 |
| -------- | ------ | -------- | -------- |
| 按范围 | 全局锁 | 整个数据库 | 全库逻辑备份 |
| 按范围 | 表级锁 | 整张表 | 表锁、MDL、意向锁、AUTO-INC 锁 |
| 按范围 | 行级锁 | 单条记录或记录间间隙 | 并发控制、解决幻读 |
| 按属性 | 共享锁 S | 读锁 | 读读共享、读写互斥 |
| 按属性 | 独占锁 X | 写锁 | 写写互斥、读写互斥 |

## 二、全局锁

### 2.1 使用方式与场景

全局锁通过以下命令实现：

```sql
-- 加全局读锁，整个库变为只读
flush tables with read lock;

-- 释放全局锁
unlock tables;
```

| 特性 | 说明 |
| ---- | ---- |
| 阻塞范围 | 阻塞所有 DML（insert/update/delete）和 DDL（alter/drop） |
| 释放时机 | `unlock tables` 或会话断开 |
| 典型场景 | 不支持事务的引擎（如 MyISAM）做全库逻辑备份 |
| 替代方案 | InnoDB 可用 `mysqldump --single-transaction`，利用 RR 事务 + MVCC 实现一致性备份 |

> **注意**：InnoDB 尽量不要用全局锁备份，因为会阻塞业务写操作；使用 `--single-transaction` 可在备份期间保持业务可写。

## 三、表级锁

### 3.1 表锁（Table Lock）

通过 `lock tables` 显式加锁，粒度大，InnoDB 中尽量避免使用：

```sql
-- 表级共享锁（读锁）
lock tables t_student read;

-- 表级独占锁（写锁）
lock tables t_student write;

-- 释放当前会话所有表锁
unlock tables;
```

| 锁类型 | 本线程限制 | 其他线程限制 |
| ------ | ---------- | ------------ |
| `read` 锁 | 只能读被锁表，不能写，也不能访问其他表 | 可读，不可写 |
| `write` 锁 | 可读写被锁表，但不能访问其他表 | 不可读、不可写 |

### 3.2 元数据锁 MDL

MDL 由 MySQL 自动加，无需显式调用：

| 操作 | MDL 类型 | 释放时机 |
| ---- | -------- | -------- |
| CRUD（select/insert/update/delete） | 读锁 | 事务提交后 |
| DDL（alter/drop 等结构变更） | 写锁 | 语句执行完成后 |

> **关键风险**：长事务持有 MDL 读锁时，若有 DDL 请求 MDL 写锁，会阻塞后续所有该表的 CRUD，导致线程池爆满。做表结构变更前应先检查并清理长事务。

### 3.3 意向锁

意向锁是表级锁，用于快速判断表中是否存在行级锁：

| 类型 | 英文 | 触发条件 |
| ---- | ---- | -------- |
| 意向共享锁 | IS | 对某行加 S 锁前，先在表级别加 IS |
| 意向独占锁 | IX | 对某行加 X 锁前，先在表级别加 IX |

- 意向锁之间互不冲突；
- 意向锁只与**表级共享锁 / 表级独占锁**（`lock tables ... read/write`）冲突；
- 作用：避免加表锁时遍历全表检查是否有行锁。

### 3.4 AUTO-INC 锁

用于保证自增主键的连续性，通过 `innodb_autoinc_lock_mode` 控制：

| 模式 | 加锁方式 | 释放时机 | 适用场景 |
| ---- | -------- | -------- | -------- |
| 0 | AUTO-INC 锁（表锁） | 插入语句执行结束后 | 严格连续 |
| 1 | 普通 insert 用轻量锁；批量 insert（如 `insert ... select`）用 AUTO-INC 锁 | 普通 insert 申请完即释放；批量 insert 语句结束后释放 | 默认模式，平衡性能与连续性 |
| 2 | 轻量锁 | 申请到自增值后立即释放 | 性能最高，但需 `binlog_format=row` 以避免主从不一致 |

> **注意**：`innodb_autoinc_lock_mode=2` 搭配 `binlog_format=statement` 可能导致主从自增 ID 不一致。

## 四、行级锁

InnoDB 的普通 `SELECT` 属于快照读，不加锁；锁定读使用以下语句：

```sql
-- 对读取记录加共享锁 S
select ... lock in share mode;

-- 对读取记录加独占锁 X
select ... for update;
```

### 4.1 共享锁与独占锁的兼容性

![S 锁与 X 锁兼容性](images/s-x-lock-compat.png)

| 已有锁 \ 申请锁 | S 锁 | X 锁 |
| --------------- | ---- | ---- |
| S 锁 | 兼容 | 互斥 |
| X 锁 | 互斥 | 互斥 |

### 4.2 Record Lock（记录锁）

锁住单条记录，有 S/X 两种类型：

```sql
begin;
select * from t_test where id = 1 for update;
```

![记录锁](images/record-lock.png)

| 已有锁 \ 申请锁 | S 型记录锁 | X 型记录锁 |
| --------------- | ---------- | ---------- |
| S 型记录锁 | 兼容 | 互斥 |
| X 型记录锁 | 互斥 | 互斥 |

### 4.3 Gap Lock（间隙锁）

锁定索引记录之间的间隙，不包含记录本身：

![间隙锁](images/gap-lock.png)

| 特性 | 说明 |
| ---- | ---- |
| 存在隔离级别 | 可重复读、串行化 |
| 作用 | 防止幻读，阻止其他事务在间隙中插入新记录 |
| 兼容性 | 间隙锁之间兼容，S 型与 X 型无区别 |
| 示例 | 间隙锁 (3, 5) 会阻止插入 id = 4 的记录 |

### 4.4 Next-Key Lock（临键锁）

Record Lock + Gap Lock 的组合，锁定范围并包含记录本身：

![临键锁](images/next-key-lock.png)

| 特性 | 说明 |
| ---- | ---- |
| 范围 | 前开后闭，如 (3, 5] |
| 作用 | 既能保护记录本身，又能阻止向记录前间隙插入新记录 |
| 兼容性 | 相同范围的 X 型 next-key lock 之间互斥 |

### 4.5 插入意向锁

插入意向锁是一种特殊的间隙锁：

| 特性 | 说明 |
| ---- | ---- |
| 触发条件 | 事务插入记录时，发现插入位置已被其他事务加了间隙锁 |
| 锁状态 | 生成锁结构并置为等待状态，直到间隙锁释放 |
| 锁级别 | 行级锁，不是意向锁 |
| 与间隙锁关系 | 同一间隙内，间隙锁与插入意向锁互斥；不在同一区间则可共存 |

## 五、锁机制对比

| 锁类型 | 粒度 | 显式/自动 | 释放时机 | 主要风险/注意 |
| ------ | ---- | --------- | -------- | -------------- |
| 全局锁 | 数据库 | 显式 | `unlock tables` / 会话结束 | 业务长时间只读，影响大 |
| 表锁 | 表 | 显式 | `unlock tables` / 会话结束 | 粒度大，并发低 |
| MDL | 表 | 自动 | 事务提交 / DDL 完成 | 长事务可导致 DDL 阻塞并引发线程爆满 |
| 意向锁 | 表 | 自动 | 随事务提交 | 仅与表锁冲突，加速表锁判断 |
| AUTO-INC 锁 | 表 | 自动 | 语句结束 / 申请到自增值后 | 模式 2 需搭配 row binlog |
| Record Lock | 行 | 显式（锁定读） | 事务提交 | 注意 S/X 兼容性 |
| Gap Lock | 行间隙 | 自动 | 事务提交 | 间隙锁之间兼容 |
| Next-Key Lock | 行 + 间隙 | 自动 | 事务提交 | RR 下防止幻读的主要手段 |

## N. 复习清单

1. **MySQL 锁按范围分为哪几类？** 全局锁、表级锁、行级锁。
2. **InnoDB 为什么推荐使用行级锁而不是表锁？** 表锁粒度大、并发低；行级锁只锁必要记录，并发高。
3. **全局锁的典型使用场景和替代方案是什么？** 用于 MyISAM 全库备份；InnoDB 可用 `mysqldump --single-transaction`。
4. **MDL 读锁和写锁分别在什么时候加？** CRUD 加 MDL 读锁，DDL 加 MDL 写锁。
5. **为什么长事务会导致 DDL 阻塞后续所有查询？** MDL 写锁等待优先级高于读锁，会阻塞后续所有该表的 CRUD。
6. **意向锁的作用是什么？** 快速判断表中是否存在行级锁，避免加表锁时遍历全表。
7. **innodb_autoinc_lock_mode=2 时需要注意什么？** 必须搭配 `binlog_format=row`，否则主从可能不一致。
8. **快照读和锁定读的区别是什么？** 普通 `SELECT` 是快照读（MVCC，不加锁）；`SELECT ... FOR UPDATE` / `LOCK IN SHARE MODE` 是锁定读。
9. **Record Lock、Gap Lock、Next-Key Lock 有什么区别？** Record Lock 锁记录；Gap Lock 锁间隙；Next-Key Lock 锁记录 + 间隙。
10. **间隙锁之间是否互斥？** 不互斥；两个事务可同时持有相同范围的间隙锁。
11. **插入意向锁是什么级别的锁？** 行级锁，是一种特殊的间隙锁，用于标识某事务等待在间隙中插入记录。
12. **RR 隔离级别下如何防止幻读？** 快照读通过 MVCC，当前读通过 next-key lock（记录锁 + 间隙锁）。
