# 什么是大语言模型（LLM）

> 一句话总结：**LLM = 用海量语料预训练、参数百亿千亿级、自回归预测下一个 token 的统一生成式模型**；它与传统 NLP 的根本差异是**任务方式、范式、能力来源**三个层面的全面重构。

---

## 0. 面试题切入

> 👔 大语言模型是什么？和传统 NLP 模型有什么区别？
>
> ❌ 错误答法：「参数多、能聊天的模型」/「端到端模型」（2015 年 LSTM 就是端到端）
>
> ✅ 正确答法：从**任务方式 / 输出范式 / 能力来源**三层讲清楚本质。

---

## 1. 传统 NLP 是怎么干活的

**流水线式**：一个任务拆成 N 步，每步一个专门模型。

| 步骤 | 模型 | 作用 |
|---|---|---|
| 1 | 分词 | 「我想退货」→ 「我 / 想 / 退货」 |
| 2 | 词性标注 | 标出代词、动词、名词 |
| 3 | 命名实体识别 | 找商品名、订单号 |
| 4 | 意图分类 | 咨询 / 投诉 / 退货 |
| 5 | 知识库匹配 | 返回预设答案 |

**三大致命问题**：
1. **错误累积传导**：分词错 → 词性错 → 实体错 → 意图错，前面一错后面全错，**无法事后补救**。
2. **OOV 问题**：训练时没见过的词（如「奥利给」「绝绝子」），分词模型直接懵。
3. **迁移成本极高**：从客服换到医疗，所有模型基本**全部重训**（实体、标注规范完全不一样）。

> 死循环：**任务越细分 → 模型越多 → 标注越贵 → 迁移越难**。

> **小结**：
> - 传统 NLP 的痛点 = **碎片化**（一任务一模型）。
> - 错一步全错是结构性问题，不是工程问题。
> - 领域迁移成本高到企业养不起几套模型团队。

## 2. BERT 时代：预训练通了一半

2018 年 Google 推出 **BERT**，核心创新是**「预训练 + 微调」两阶段范式**。

### 2.1 预训练阶段

| 任务 | 做法 | 目的 |
|---|---|---|
| **MLM**（Masked Language Model） | 随机遮掉 15% 词，让模型猜 | 学**字词级**上下文表示 |
| **NSP**（Next Sentence Prediction） | 给两句话，判断是否连续 | 学**句子级**连贯关系 |

数据来源：维基百科 + 图书数据，**约 33 亿词**（当时已是海量），**无需人工标注**。

### 2.2 微调阶段

下游任务接一个「任务头」（分类头 / 序列标注头 / QA 头），用少量标注数据微调。

### 2.3 BERT 为什么"通了一半"

| 维度 | 状态 |
|---|---|
| **特征通用** | ✅ 一个 BERT 服务多个下游任务 |
| **任务统一** | ❌ 不同任务仍需不同的微调副本 |

**两个根因**：
1. **输出是「表示」不是「文本」**：每一层输出 token 向量，要做分类得接分类头、做 NER 得接序列标注头、做 QA 得接抽取式头 → **一个 BERT 派生出十几个微调副本**。
2. **不擅长生成**：MLM 目标是"填空"，没学过连续生成长文本 → 翻译、写作、对话做不了。

> 评价：**BERT 把"理解"做到极致，但"生成"是短板**。把"预训练 + 大规模无标注数据"这条路验证对了，但没把所有任务收归到同一个接口下。

> **小结**：
> - BERT = 「**预训练 + 微调**」范式的开创者。
> - 解决了"特征通用"，没解决"任务统一"。
> - 理解强，生成弱。

## 3. LLM 的本质：把任务收编成"预测下一个 token"

### 3.1 训练目标：CLM（Causal Language Modeling）

> 给一段文本，模型从左到右**自回归**（Autoregressive）地预测下一个 token。

```
训练样本：「我喜欢吃苹果」
看到「我」      → 预测「喜」
看到「我喜」    → 预测「欢」
看到「我喜欢」  → 预测「吃」
看到「我喜欢吃」→ 预测「苹果」
```

**GPT / Claude / Qwen / DeepSeek 都是这个范式**：自回归 + 因果掩码。

### 3.2 威力：所有任务都变成"Prompt + 续写"

| 任务 | Prompt 写法 | 模型输出 |
|---|---|---|
| 翻译 | `把下面翻译成英文：我喜欢你 ->` | `I like you` |
| 分类 | `这条评论正面还是负面？『这家店太黑了』 -> 答：` | `负面` |
| 总结 | `用一句话总结：xxxxxx -> 总结：` | 总结内容 |
| 写代码 | `写一个 Python 函数返回斐波那契前 N 项 -> def` | 完整代码 |

> **关键洞察**：**不需要为每个任务训不同模型**，只要在 Prompt 里换说法，一个模型就能切换工作模式。

### 3.3 为什么这个简单目标能学到这么多

**两个杠杆**：

1. **数据杠杆**：CLM 不需要任何人工标注，互联网所有文本都是合格训练数据。
   - GPT-3 = 3000 亿 token
   - Llama 3 = **15 万亿 token**
   - 对比 BERT = 几十亿词，**规模翻了几千倍**。

2. **模型杠杆**：参数量从 BERT 0.3B → GPT-3 175B → 更大闭源模型。
   - 要准确预测下一个 token，**必须**学到语法、事实、推理。
   - 例：预测「北京是中国的____」→ 必须知道"北京是首都"。

> ⚠️ **面试避坑**：GPT-4 等闭源模型参数量**官方未公开**，别说"万亿级"确定事实。

### 3.4 副产物：In-Context Learning（ICL）

> 在 Prompt 里给几个例子，模型**不更新参数**就能学会新任务。

```
苹果 -> apple
香蕉 -> banana
草莓 -> strawberry
橘子 ->   ← 模型自动输出 "orange"
```

> 这种能力是 GPT-3 之后才被业界发现，**也是 Prompt Engineering 这门工程学科诞生的基础**。

> **小结**：
> - LLM 本质 = **CLM 自回归预测下一个 token**。
> - 一个训练目标覆盖所有 NLP 任务，靠的是 **Prompt + 续写** 统一接口。
> - 数据规模和参数规模是两大杠杆。
> - ICL 是 Prompt Engineering 诞生的根基。

## 4. 涌现能力：量变到质变的关键

> **涌现（Emergent Abilities）**：某项能力在小模型上几乎看不到，规模到临界点后**突然**表现出来。

⚠️ **学术 caveat**：有研究认为部分"突然出现"来自**评测指标离散性**（如 exact match），换连续指标后曲线可能平滑。面试时可以说"工程上能观察到的能力跃迁，学术上对是否测量假象还有争议"。

### 4.1 三个真实涌现案例

| 案例 | 小模型表现 | 临界点 | 大模型表现 |
|---|---|---|---|
| **多步算术**（5 步应用题） | <8B 准确率 ≈ 0；62B ≈ 5% | **540B**（PaLM） | 准确率突然跳到 **60%** |
| **In-Context Learning** | GPT-2 1.5B 完全无此能力 | **~100B** | GPT-3 175B 突然出现 ICL |
| **跨语言迁移** | — | 临界点不明确 | GPT-3（英文 92%）能直接处理中文、日文、阿拉伯语、冰岛语 |

### 4.2 为什么涌现：Scaling Law（缩放定律）

> 模型规模、训练数据量、训练算力三者存在**可预测的幂律关系**：三者按比例放大，损失值沿幂律曲线下降。

- **OpenAI 2020 年论文**：首次提出 Scaling Law。
- **DeepMind Chinchilla**：给出更精细比例——**参数和数据按 1:20 配**（如 70B 参数配 1.4T token）。

### 4.3 涌现不是"越大越好"

> **超过某个规模后，单纯堆参数的边际收益递减**。
> 
> 现在的趋势：**参数不必最大，但数据要够多、算力要够花**。
> 
> 例子：**Llama 3 8B 用 15T token 训，效果比早期 GPT-3 175B 还强**——数据规模超过参数规模的回报。

> **小结**：
> - 涌现 = 规模到临界点后**能力跃迁**。
> - 三大典型：多步算术 / ICL / 跨语言迁移。
> - Scaling Law 指导：参数 × 数据 × 算力 按比例放大。
> - 现实策略：**数据 > 单纯堆参数**。

## 5. 三个本质区别总结

| 维度 | 传统 NLP | LLM |
|---|---|---|
| **任务方式** | 一任务一模型，pipeline 串联 | 一个模型干所有事，Prompt 统一接口 |
| **输出范式** | 判别式（输出标签/概率） | 生成式（输出文本） |
| **能力来源** | 显式监督（喂什么学什么） | 大规模预训练 + 涌现（学到没教过的能力） |

### 5.1 三个变化的影响

| 层面 | 变化 | 对工程团队的影响 |
|---|---|---|
| **工程层面** | 拼装积木 → 统一系统 | 团队结构从「N 个模型组」变「一个大模型组」；部署从 N 个模型变「一个 base 复用多场景」；迭代跟着 base model 升级走 |
| **范式层面** | 分类器思维 → 生成器思维 | 用户交互从「点选项」变「打字聊天」；产品思考从「穷举意图」变「写好 Prompt」；评估指标从「准确率/召回率」变「LLM-as-Judge 评分」 |
| **能力层面** | 显式能力 → 涌现能力 | 过去加能力要标新数据；现在加能力只要改 Prompt；**能力上限不再被人工标注规模锁死** |

> **小结**：
> - 这不是「又出了一个新模型」，**是整个 NLP 行业工作方式被重写**。
> - 工程师技能栈从「词法分析/特征工程/调参」变「Prompt Engineering / RAG / Agent / 对齐微调」。
> - "大模型应用工程师"岗位工资高出一截——**老技能不够用了**。

## 6. 复习清单

1. **LLM 的本质定义？** 海量语料预训练 + 百亿千亿参数 + 自回归预测下一个 token 的统一生成式模型。
2. **传统 NLP 三大致命问题？** 错误累积传导、OOV 问题、迁移成本极高。
3. **BERT 解决了什么没解决什么？** 解决"特征通用"（一个 BERT 服务多任务），没解决"任务统一"（不同任务仍需不同微调副本）。
4. **BERT 两个根因？** 输出是"表示"不是"文本"（要接任务头）；不擅长生成（MLM 是填空题不是续写）。
5. **CLM 训练目标？** Causal Language Modeling，自回归预测下一个 token。
6. **为什么 CLM 能覆盖所有任务？** 翻译/分类/总结/写代码都能用「Prompt + 续写」表达。
7. **数据规模和参数规模哪个更重要？** 数据规模回报更高（Llama 3 8B + 15T token > 早期 GPT-3 175B）。
8. **In-Context Learning 是什么？** 在 Prompt 里给几个例子，模型不更新参数就学会新任务。
9. **涌现能力的定义？** 某能力在小模型上看不见，规模到临界点后突然表现出来的现象。
10. **Scaling Law 关键结论？** 模型规模、训练数据、算力按比例放大，损失沿幂律下降；Chinchilla 1:20 比例。
11. **面试官可能挖的坑？** "GPT-4 参数量多少？"——闭源模型不要瞎猜，说"百亿到千亿级，外界只能估计"。
12. **三个本质区别？** 任务方式（pipeline → 统一 Prompt）、输出范式（判别式 → 生成式）、能力来源（显式监督 → 大规模预训练 + 涌现）。
