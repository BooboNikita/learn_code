# 数智工程师 — 笔试考点与复习指南

> 面向大型能源央企人力资源共享团队 · 数智工程师岗的社招笔试准备
> 笔试模式：机考，满分 100 分，时间 120 分钟
> 题型参考：单选题 + 多选题 + 判断题 + 简答/论述题（社招侧重专业深度）

---

## 一、笔试题型与分值预估

社招笔试通常分为**综合知识**和**专业知识**两部分。

| 模块 | 占比 | 内容范围 |
|------|------|----------|
| 综合知识 | ~30-40% | 行测（言语/数理/逻辑）、企业常识、时政 |
| 专业知识 | ~60-70% | AI 与大数据基础、数字化项目、RPA 与低代码、数据治理、HR 业务场景 |

## 二、考点一：人工智能基础

> JD 要求：了解人工智能、大数据等技术；研究 AI 在智能招聘、人才画像、绩效预测、薪酬分析等场景的应用

### 2.1 AI 基本概念

- AI / ML / DL 的层级关系
- 机器学习类型：监督学习、无监督学习、强化学习
- 常见算法分类：分类（SVM/决策树/随机森林）、回归（线性/逻辑回归）、聚类（K-Means/DBSCAN）

### 2.2 大数据基础

- 大数据 5V 特征：Volume、Velocity、Variety、Veracity、Value
- 数据处理流程：采集 → 存储 → 计算 → 分析 → 可视化
- Hadoop 生态组件：HDFS（存储）、MapReduce/Spark（计算）、Hive（数仓）、Kafka（消息）

### 2.3 大模型基础（了解级别）

- LLM 基本概念：预训练、微调、Prompt Engineering
- 常见大模型：ChatGPT、文心一言、通义千问、DeepSeek
- 大模型在企业中的应用场景：智能客服、文档生成、知识问答、代码辅助

### 2.4 AI 在 HR 领域的应用场景

| 场景 | AI 能力 | 典型方案 |
|------|---------|----------|
| 智能招聘 | 简历解析、人岗匹配、AI 筛选 | NLP 文本分类、语义匹配 |
| 人才画像 | 多维特征建模、能力评估 | 用户画像、聚类分析 |
| 绩效预测 | 历史数据分析、趋势预测 | 回归模型、时序分析 |
| 薪酬分析 | 市场对标、内部公平性 | 统计分析、异常检测 |
| 员工服务 | 自助问答、流程引导 | 智能客服、对话机器人 |

## 三、考点二：RPA 与低代码平台

> JD 要求：了解 RPA 流程自动化、低代码平台等技术；主导智能应用场景的设计与开发

### 3.1 RPA（机器人流程自动化）

| 考点 | 内容 |
|------|------|
| 核心概念 | 模拟人工操作，自动化执行规则明确的重复性任务 |
| 主要厂商 | UiPath、Automation Anywhere、Blue Prism、艺赛旗（国产） |
| 适用场景 | 数据录入、报表生成、跨系统数据搬运、对账核对 |
| 不适用场景 | 需要复杂判断、非结构化数据处理 |
| 与 AI 结合 | IPA（智能流程自动化）= RPA + AI + OCR + NLP |

### 3.2 RPA 开发基础

- RPA 架构：Studio（开发）→ Orchestrator（调度）→ Robot（执行）
- 元素定位：选择器（Selector）、图像识别、OCR
- 异常处理：重试机制、业务异常 vs 系统异常
- 数据处理：Excel 操作、数据库读写、邮件收发

### 3.3 低代码/零代码平台

| 考点 | 内容 |
|------|------|
| 核心价值 | 可视化拖拽开发，降低开发门槛，快速构建应用 |
| 主要平台 | 钉钉宜搭、飞书多维表格、简道云、明道云、Appian、Mendix |
| 适用场景 | 表单审批流程、数据看板、简易业务系统 |
| 局限性 | 复杂业务逻辑受限、性能瓶颈、定制化能力有限 |
| 选型考量 | 易用性、集成能力、扩展性、安全合规 |

### 3.4 RPA + 低代码协同

- 低代码搭建前端交互界面，RPA 处理后台跨系统操作
- 典型场景：员工自助入职（低代码表单 + RPA 多系统联动创建账号）

## 四、考点三：数据治理与主数据管理

> JD 要求：熟悉数据治理、主数据管理；推进数据质量治理、数据安全合规

### 4.1 数据治理基础

| 考点 | 内容 |
|------|------|
| 数据治理定义 | 对数据资产管理行使权力和控制的活动集合（规划、监督、控制） |
| 治理体系 | 组织架构、制度规范、技术工具、流程执行 |
| 数据标准 | 命名规范、编码规则、字典定义、分类体系 |
| 元数据管理 | 业务元数据、技术元数据、操作元数据 |

### 4.2 数据质量管理

- 数据质量六维度：完整性、准确性、一致性、及时性、唯一性、有效性
- 数据质量治理流程：定义规则 → 质量检测 → 问题分析 → 整改优化 → 持续监控
- 数据清洗方法：去重、补全、标准化、纠错

### 4.3 主数据管理（MDM）

- 主数据定义：跨业务共享的、核心的基础数据（人员、组织、岗位、供应商等）
- 主数据 vs 参考数据 vs 交易数据的区别
- MDM 实施模式：注册制、合并制、集中制、共存制
- 主数据管理的价值：统一数据口径、消除数据孤岛、支撑分析决策

### 4.4 数据安全与合规

- **《数据安全法》**：数据分类分级、安全评估、跨境传输
- **《个人信息保护法》**：知情同意、最小必要、目的限制
- HR 数据特殊合规：敏感个人信息（薪资/绩效/健康状况）的额外保护要求
- 数据权限管理：RBAC（基于角色的访问控制）、数据脱敏、审计日志

## 五、考点四：数字化项目管理

> JD 要求：全周期管理信息化项目，协调多方角色；具备产品思维和项目管理能力

### 5.1 项目管理基础

| 考点 | 内容 |
|------|------|
| 项目生命周期 | 启动 → 规划 → 执行 → 监控 → 收尾 |
| 常见方法论 | 瀑布模型、Scrum 敏捷、看板（Kanban） |
| 敏捷实践 | Sprint 迭代、每日站会、需求 Backlog、回顾会议 |
| 范围管理 | 需求调研、WBS 分解、变更控制 |
| 风险管理 | 风险识别 → 评估 → 应对策略（规避/转移/减轻/接受） |

### 5.2 需求工程

- 需求调研方法：访谈、问卷、观察、工作坊
- 需求文档：BRD（业务需求）→ PRD（产品需求）→ SRD（系统需求）
- 需求优先级：MoSCoW 法则（Must/Should/Could/Won't）、Kano 模型
- 需求跟踪矩阵：需求 → 设计 → 开发 → 测试 → 上线的全链路追踪

### 5.3 系统架构设计

- **业务架构**：业务流程建模、组织架构、价值链分析
- **应用架构**：单体 vs 微服务，应用分层（展示/业务/数据）
- **数据架构**：数据仓库、数据湖、ETL 流程、数据分层（ODS/DWD/DWS/ADS）
- **技术架构**：部署架构（云/私有化/混合）、高可用设计

### 5.4 产品思维

- 从业务痛点提炼产品需求
- MVP（最小可行产品）策略
- 模块化设计、可复用组件
- 用户体验要素：易用性、稳定性、业务适配性
- 指标体系：用户活跃度、任务完成率、系统响应时间

## 六、考点五：HR 共享服务与人力资源业务

> JD 要求：推动人力资源共享业务数字化与智能化转型；理解招聘、入转调离、薪酬核算等核心模块

### 6.1 HR 共享服务模式

| 考点 | 内容 |
|------|------|
| SSC 定位 | 将 HR 事务性工作集中标准化处理，提升效率、降低成本 |
| 三支柱模型 | COE（专家中心）/ HRBP（业务伙伴）/ SSC（共享服务中心） |
| 服务范围 | 招聘执行、入转调离、薪酬核算、社保公积金、员工咨询、报表统计 |
| 转型方向 | 从事务处理 → 数据分析 → 智能决策，从成本中心 → 价值中心 |

### 6.2 HR 核心业务模块

| 模块 | 核心流程 | 数字化痛点 |
|------|----------|------------|
| 招聘管理 | 需求发布 → 简历筛选 → 面试安排 → 录用审批 → 入职 | 简历量大筛选效率低、跨系统数据不同步 |
| 入转调离 | 入职办理 → 试用期管理 → 转正/调岗/离职 | 流程繁琐、多系统重复录入 |
| 薪酬管理 | 考勤汇总 → 薪资计算 → 社保公积金 → 个税 → 发放 | 数据来源多、核算规则复杂、合规要求高 |
| 绩效管理 | 目标设定 → 过程跟踪 → 考核评估 → 结果应用 | 评估标准不统一、过程难追踪 |
| 员工服务 | 政策咨询 → 证明开具 → 流程申请 → 投诉处理 | 响应慢、知识分散、体验差 |

### 6.3 HR 数字化转型趋势

- 人效分析：人均产出、编制优化、用工成本预测
- 数字化员工体验（DEX）：员工 App、自助服务、智能助手
- 人才管理：能力模型、发展路径、继任计划
- 组织分析：组织健康度、离职预测、敬业度分析

## 九、复习优先级与策略

### 按重要度排序

| 优先级 | 考点 | 原因 |
|--------|------|------|
| **P0** | 数据治理 + 主数据 | JD 反复强调，岗位核心职责 |
| **P0** | RPA + 低代码平台 | 智能应用开发直接相关 |
| **P0** | HR 共享业务 | 必须理解业务才能做数字化转型 |
| **P1** | AI 基础 + 大数据 | "了解"级别，不会考太深 |
| **P1** | 数字化项目管理 | 产品经理/项目负责人能力验证 |
| **P1** | 数据库 + 系统集成 | 系统设计能力的基础 |
| **P2** | 行测 + 时政 | 综合知识，拉分项 |

### 复习建议

1. **数据治理**（最重要）：重点掌握数据质量六维度、主数据管理概念、数据安全法规，这是岗位日常工作的核心
2. **RPA + 低代码**：理解 RPA 基本架构和适用场景，了解主流低代码平台，能说出 HR 场景下的具体应用案例
3. **HR 业务理解**：熟悉 SSC 三支柱模型、招聘/薪酬/入转调离等核心流程的痛点和数字化改进方向
4. **AI 部分**：不需要深入算法推导，重点掌握 AI 在 HR 领域的应用场景和大模型的基本使用方式
5. **项目管理**：了解敏捷开发流程、需求分析方法，能从业务痛点出发设计数字化方案
6. **综合知识**：刷行测题保持手感，关注近期数据安全、AI 治理相关法规政策
